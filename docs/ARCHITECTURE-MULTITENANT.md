# Architecture Multi-tenant Hierarchique — Ticket 014

> Version : 1.0  
> Date : 2026-06-25  
> Ticket : 014 — Architecture Multi-tenant Hierarchique et Securisation des Donnees  
> Statut : Fondations implementees

---

## 1. Contexte et Objectif

EBH Pilot evolue d une application mono-tenant vers une plateforme SaaS generique. Ce ticket pose les fondations permettant a plusieurs entreprises clientes (tenants) de cohabiter sur la meme infrastructure, avec une isolation stricte des donnees et une hierarchie managériale native.

### Objectifs du ticket

- Definir la structure organisationnelle multi-tenant (Organization / Region / Sector / Site)
- Mettre en place la hierarchie manager/subordonnes sur le modele User
- Introduire un systeme de roles metier (CEO → SALES_REP)
- Associer toutes les donnees metier a un proprietaire (ownerId)
- Preparer les mecanismes de visibilite et d isolation
- Documenter les impacts futurs

---

## 2. Analyse de l Existant

### Modeles avant Ticket 014

| Modele | Champs cles | Observations |
|--------|-------------|---------------|
| User | id, email, name, authId | Pas de role, pas d organisation |
| Prospect | userId → User | Lie a un user mais pas de notion de proprietaire |
| Product | userId → User | Idem |
| ImportJob | userId → User (nullable) | Idem |

### Problemes identifies

- Pas de notion de tenant : toutes les donnees sont partagees implicitement
- Pas de hierarchie entre utilisateurs
- Pas de roles : impossible de distinguer un CEO d un commercial
- userId utilisé a la fois comme propriétaire et comme createur — ambigu
- Aucune structure geographique (regions, secteurs, sites)

## 3. Schema Retenu

### 3.1 Structure Organisationnelle

```
Organization (tenant SaaS)
  └── Region
          └── Sector
                        └── Site
                                            └── User
                                            ```

                                            ### 3.2 Hierarchie Managériale

                                            ```
                                            CEO
                                              └── REGIONAL_DIRECTOR (rattache a une Region)
                                                      └── SECTOR_DIRECTOR (rattache a un Sector)
                                                                    └── SITE_DIRECTOR (rattache a un Site)
                                                                                        └── SALES_MANAGER (rattache a un Site)
                                                                                                                  ├── SALES_REP A
                                                                                                                                            ├── SALES_REP B
                                                                                                                                                                      └── SALES_REP C
                                                                                                                                                                      ```
                                                                                                                                                                      
                                                                                                                                                                      ### 3.3 Nouveaux Modeles Prisma
                                                                                                                                                                      
                                                                                                                                                                      #### Organization
                                                                                                                                                                      
                                                                                                                                                                      Représente un client SaaS (un tenant). Chaque organisation est independante.
                                                                                                                                                                      
                                                                                                                                                                      Champs cles : id, name, slug (unique), logoUrl, primaryColor, secondaryColor, isActive
                                                                                                                                                                      
                                                                                                                                                                      Le slug permet un acces URL propre (ex: `acme-corp`) et l identification du tenant.
                                                                                                                                                                      Les champs de couleurs (primaryColor, secondaryColor) preparent la personnalisation visuelle.
                                                                                                                                                                      
                                                                                                                                                                      #### Region
                                                                                                                                                                      
                                                                                                                                                                      Unite geographique appartenant a une organisation. Ex : Region Nord, Ile-de-France.
                                                                                                                                                                      
                                                                                                                                                                      Relation : Organization (1) → Region (N)
                                                                                                                                                                      
                                                                                                                                                                      #### Sector
                                                                                                                                                                      
                                                                                                                                                                      Sous-unite geographique appartenant a une region. Ex : Secteur Paris, Secteur Lyon.
                                                                                                                                                                      
                                                                                                                                                                      Relation : Region (1) → Sector (N)
                                                                                                                                                                      
                                                                                                                                                                      #### Site
                                                                                                                                                                      
                                                                                                                                                                      Point de vente ou agence physique appartenant a un secteur.
                                                                                                                                                                      
                                                                                                                                                                      Relation : Sector (1) → Site (N)
                                                                                                                                                                      
                                                                                                                                                                      ### 3.4 Modele User Enrichi
                                                                                                                                                                      
                                                                                                                                                                      Le modele User a ete significativement enrichi :
                                                                                                                                                                      
                                                                                                                                                                      - `role: UserRole` — role dans la hierarchie (enum)
                                                                                                                                                                      - `isActive: Boolean` — permet de desactiver un compte sans suppression
                                                                                                                                                                      - `organizationId` — rattachement au tenant
                                                                                                                                                                      - `regionId`, `sectorId`, `siteId` — rattachements geographiques selon le role
                                                                                                                                                                      - `managerId` / `subordinates` — hierarchie native (auto-reference)
                                                                                                                                                                      
                                                                                                                                                                      ### 3.5 Propriété des Donnees (ownerId)
                                                                                                                                                                      
                                                                                                                                                                      Toutes les donnees metier sont maintenant rattachees a un proprietaire :
                                                                                                                                                                      
                                                                                                                                                                      | Modele | Avant | Apres |
                                                                                                                                                                      |--------|-------|-------|
                                                                                                                                                                      | Prospect | userId (User) | ownerId (User, relation "ProspectOwner") |
                                                                                                                                                                      | Product | userId (User) | ownerId (User, relation "ProductOwner") |
                                                                                                                                                                      | ImportJob | userId nullable | ownerId nullable (relation "ImportJobOwner") |
                                                                                                                                                                      
                                                                                                                                                                      **Pourquoi ownerId plutot que userId ?**
                                                                                                                                                                      
                                                                                                                                                                      `userId` etait ambigu — signifiait a la fois "createur" et "proprietaire". Le nouveau champ `ownerId` est semantiquement clair : il designe le commercial responsable de la donnee, qui peut etre different du createur (ex : import batch par un manager).

                                                                                                                                                                      ## 4. Systeme de Roles
                                                                                                                                                                      
                                                                                                                                                                      ### 4.1 Definition des Roles
                                                                                                                                                                      
                                                                                                                                                                      | Role | Niveau | Description |
                                                                                                                                                                      |------|--------|-------------|
                                                                                                                                                                      | CEO | 6 | Direction generale. Voit toutes les donnees de l organisation. |
                                                                                                                                                                      | REGIONAL_DIRECTOR | 5 | Dirige une region. Voit toutes les donnees de sa region. |
                                                                                                                                                                      | SECTOR_DIRECTOR | 4 | Dirige un secteur. Voit toutes les donnees de son secteur. |
                                                                                                                                                                      | SITE_DIRECTOR | 3 | Dirige un site. Voit toutes les donnees de son site. |
                                                                                                                                                                      | SALES_MANAGER | 2 | Responsable commercial. Voit ses donnees + celles de son equipe. |
                                                                                                                                                                      | SALES_REP | 1 | Commercial. Voit uniquement ses propres donnees. |
                                                                                                                                                                      
                                                                                                                                                                      ### 4.2 Responsabilites par Role
                                                                                                                                                                      
                                                                                                                                                                      **CEO**
                                                                                                                                                                      - Acces complet a toutes les donnees de l organisation
                                                                                                                                                                      - Peut consulter, modifier, et supprimer toutes les ressources
                                                                                                                                                                      - Acces a tous les KPI futurs (individuel, equipe, site, secteur, region, organisation)
                                                                                                                                                                      - Gestion des utilisateurs, des regions, des secteurs et des sites
                                                                                                                                                                      
                                                                                                                                                                      **REGIONAL_DIRECTOR**
                                                                                                                                                                      - Gere plusieurs secteurs au sein d une region
                                                                                                                                                                      - Voit tous les prospects, produits et indicateurs de sa region
                                                                                                                                                                      - Peut creer et gerer les secteurs et sites de sa region
                                                                                                                                                                      - Responsable des objectifs regionaux
                                                                                                                                                                      
                                                                                                                                                                      **SECTOR_DIRECTOR**
                                                                                                                                                                      - Gere plusieurs sites au sein d un secteur
                                                                                                                                                                      - Voit tous les prospects, produits et indicateurs de son secteur
                                                                                                                                                                      - Peut creer et gerer les sites de son secteur
                                                                                                                                                                      - Responsable des objectifs sectoriels
                                                                                                                                                                      
                                                                                                                                                                      **SITE_DIRECTOR**
                                                                                                                                                                      - Gere un site specifique
                                                                                                                                                                      - Voit tous les prospects, produits et indicateurs de son site
                                                                                                                                                                      - Peut gerer les utilisateurs de son site
                                                                                                                                                                      - Responsable des objectifs du site
                                                                                                                                                                      
                                                                                                                                                                      **SALES_MANAGER**
                                                                                                                                                                      - Responsable d une equipe de commerciaux
                                                                                                                                                                      - Voit ses propres donnees + les donnees de ses subordonnes directs
                                                                                                                                                                      - Peut assigner des prospects a ses commerciaux
                                                                                                                                                                      - Responsable des objectifs d equipe
                                                                                                                                                                      
                                                                                                                                                                      **SALES_REP**
                                                                                                                                                                      - Commercial de terrain
                                                                                                                                                                      - Voit et modifie uniquement ses propres donnees
                                                                                                                                                                      - Cree des prospects et produits en son nom
                                                                                                                                                                      - Responsable de ses objectifs individuels

                                                                                                                                                                      ## 5. Hierarchie Manager/Subordonnes
                                                                                                                                                                      
                                                                                                                                                                      ### 5.1 Principe
                                                                                                                                                                      
                                                                                                                                                                      La hierarchie est implementee via une auto-reference sur le modele User :
                                                                                                                                                                      
                                                                                                                                                                      ```prisma
                                                                                                                                                                      model User {
                                                                                                                                                                        managerId    String?
                                                                                                                                                                          manager      User?  @relation("UserHierarchy", fields: [managerId], references: [id])
                                                                                                                                                                            subordinates User[] @relation("UserHierarchy")
                                                                                                                                                                            }
                                                                                                                                                                            ```
                                                                                                                                                                            
                                                                                                                                                                            ### 5.2 Regles
                                                                                                                                                                            
                                                                                                                                                                            - Un utilisateur peut avoir zero ou un manager (`managerId` nullable)
                                                                                                                                                                            - Un utilisateur peut avoir zero ou plusieurs subordonnes (`subordinates[]`)
                                                                                                                                                                            - La hierarchie est directe (1 niveau de relation) : pour remonter toute la chaine, il faut une requete recursive
                                                                                                                                                                            - Le CEO n a pas de manager (managerId = null)
                                                                                                                                                                            - Un SALES_REP n a pas de subordonnes
                                                                                                                                                                            
                                                                                                                                                                            ### 5.3 Exemple Concret
                                                                                                                                                                            
                                                                                                                                                                            ```
                                                                                                                                                                            Alice (CEO, managerId: null)
                                                                                                                                                                              └── Bob (REGIONAL_DIRECTOR, managerId: Alice.id)
                                                                                                                                                                                      └── Carol (SECTOR_DIRECTOR, managerId: Bob.id)
                                                                                                                                                                                                    └── David (SITE_DIRECTOR, managerId: Carol.id)
                                                                                                                                                                                                                        └── Eve (SALES_MANAGER, managerId: David.id)
                                                                                                                                                                                                                                                  ├── Frank (SALES_REP, managerId: Eve.id)
                                                                                                                                                                                                                                                                            └── Grace (SALES_REP, managerId: Eve.id)
                                                                                                                                                                                                                                                                            ```

                                                                                                                                                                                                                                                                            ## 6. Isolation des Donnees
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 6.1 Module dataIsolation.ts
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Le fichier `apps/api/src/middleware/dataIsolation.ts` expose les fondations :
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            **ROLE_VISIBILITY** : mappe chaque role a son scope de visibilite
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            | Role | Scope |
                                                                                                                                                                                                                                                                            |------|-------|
                                                                                                                                                                                                                                                                            | CEO | ORGANIZATION |
                                                                                                                                                                                                                                                                            | REGIONAL_DIRECTOR | REGION |
                                                                                                                                                                                                                                                                            | SECTOR_DIRECTOR | SECTOR |
                                                                                                                                                                                                                                                                            | SITE_DIRECTOR | SITE |
                                                                                                                                                                                                                                                                            | SALES_MANAGER | TEAM |
                                                                                                                                                                                                                                                                            | SALES_REP | PERSONAL |
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            **buildOwnerFilter(user)** : construit le filtre Prisma WHERE selon le role
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ```typescript
                                                                                                                                                                                                                                                                            // SALES_REP :
                                                                                                                                                                                                                                                                            { ownerId: user.id }
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            // SALES_MANAGER :
                                                                                                                                                                                                                                                                            { ownerId: { in: [user.id, ...subordinateIds] } }
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            // SITE_DIRECTOR :
                                                                                                                                                                                                                                                                            { owner: { siteId: user.siteId } }
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            // CEO :
                                                                                                                                                                                                                                                                            { owner: { organizationId: user.organizationId } }
                                                                                                                                                                                                                                                                            ```
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            **canMutate(currentUser, resourceOwnerId)** : retourne true/false pour UPDATE/DELETE
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            **assertCanMutate(currentUser, resourceOwnerId)** : lance une erreur 403 si non autorise
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            **requireRole(minimumRole)** : middleware Express pour proteger les routes par role
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 6.2 Regles CREATE
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Lors de la creation d une ressource :
                                                                                                                                                                                                                                                                            1. Recuperer l utilisateur authentifie (via authId Supabase)
                                                                                                                                                                                                                                                                            2. Associer automatiquement `ownerId = user.id`
                                                                                                                                                                                                                                                                            3. L utilisateur ne peut pas specifier un `ownerId` different du sien (sauf roles superieurs)
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 6.3 Regles READ
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Appliquer `buildOwnerFilter(user)` comme clause WHERE sur toute requete `findMany`.
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 6.4 Regles UPDATE / DELETE
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Appeler `assertCanMutate(currentUser, resource.ownerId)` avant toute modification.

                                                                                                                                                                                                                                                                            ## 8. Choix Techniques
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### Pourquoi des relations optionnelles pour organizationId, regionId, etc. ?
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Les champs geographiques sont nullable pour permettre :
                                                                                                                                                                                                                                                                            - La migration des utilisateurs existants sans donnees de migration
                                                                                                                                                                                                                                                                            - Les comptes systeme ou administrateurs sans rattachement geographique
                                                                                                                                                                                                                                                                            - Une introduction progressive lors du deploiement
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### Pourquoi conserver authId separé de id ?
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            - `id` est l identifiant Prisma (UUID interne)
                                                                                                                                                                                                                                                                            - `authId` est l identifiant Supabase Auth (`auth.users.id`)
                                                                                                                                                                                                                                                                            Cette separation permet de dissocier l identite d authentification de l identite metier, facilitant les migrations futures.
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### Pourquoi des named relations (@relation) sur Prospect, Product, ImportJob ?
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Prisma exige des noms de relations explicites lorsque deux modeles ont plusieurs relations entre eux. Les noms choisis (`ProspectOwner`, `ProductOwner`, `ImportJobOwner`) sont semantiquement clairs.
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### Pourquoi ne pas utiliser Row Level Security (RLS) Supabase directement ?
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Le RLS Supabase est une excellente option pour une securite au niveau base de donnees, mais :
                                                                                                                                                                                                                                                                            - Il necessite une configuration par table dans Supabase
                                                                                                                                                                                                                                                                            - L acces au contexte utilisateur necessite la propagation du JWT
                                                                                                                                                                                                                                                                            - La logique actuelle via Prisma + middleware offre plus de flexibilite pour les requetes complexes
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Le RLS pourra etre ajoute en couche supplementaire dans un ticket dedie.

                                                                                                                                                                                                                                                                            ## 9. Impacts Futurs
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 9.1 Reporting
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            La structure en place permet les KPI a chaque niveau :
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            - **KPI individuels** : filtrer par `ownerId = user.id`
                                                                                                                                                                                                                                                                            - **KPI equipe** : filtrer par `ownerId IN subordinateIds`
                                                                                                                                                                                                                                                                            - **KPI site** : filtrer par `owner.siteId = site.id`
                                                                                                                                                                                                                                                                            - **KPI secteur** : filtrer par `owner.sector.id = sector.id`
                                                                                                                                                                                                                                                                            - **KPI region** : filtrer par `owner.region.id = region.id`
                                                                                                                                                                                                                                                                            - **KPI organisation** : filtrer par `owner.organizationId = org.id`
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 9.2 Application Mobile
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Les donnees necessaires a l app mobile sont accessibles via l API existante :
                                                                                                                                                                                                                                                                            - Challenges commerciaux : agregation des donnees par equipe avec comparaison
                                                                                                                                                                                                                                                                            - To-do list : nouveau modele Task a rattacher a ownerId
                                                                                                                                                                                                                                                                            - Activite terrain : evenements lies a Prospect avec geolocalisation optionnelle
                                                                                                                                                                                                                                                                            - Notifications : nouveau modele Notification a rattacher a userId
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 9.3 SaaS Multi-tenant
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            Pour le passage en production SaaS :
                                                                                                                                                                                                                                                                            - Ajouter la validation `organizationId` sur chaque requete API
                                                                                                                                                                                                                                                                            - Implementer le RLS Supabase pour une securite en profondeur
                                                                                                                                                                                                                                                                            - Creer un systeme d invitation pour rejoindre une organisation
                                                                                                                                                                                                                                                                            - Ajouter la gestion des abonnements (modele Subscription)
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ### 9.4 Personnalisation Avancée
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            - Endpoint `GET /organization/settings` pour recuperer logo + couleurs
                                                                                                                                                                                                                                                                            - Theming dynamique sur le frontend via CSS variables
                                                                                                                                                                                                                                                                            - Upload du logo vers Supabase Storage

                                                                                                                                                                                                                                                                            Lors de l execution de la premiere migration :
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ```bash
                                                                                                                                                                                                                                                                            npx prisma migrate dev --name add-multitenant-hierarchy

                                                                                                                                                                                                                                                                            Impact sur les donnees existantes :
                                                                                                                                                                                                                                                                            - Les champs `ownerId` sur Prospect et Product sont `NOT NULL` — il faudra migrer les donnees existantes (`ownerId = userId`) avant ou pendant la migration
                                                                                                                                                                                                                                                                            - Les nouveaux champs (organizationId, regionId, etc.) sont nullable → pas d impact bloquant

                                                                                                                                                                                                                                                                            **Script de migration de donnees recommande :**
                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                            ```sql
                                                                                                                                                                                                                                                                            -- Avant de lancer prisma migrate
                                                                                                                                                                                                                                                                            UPDATE "Prospect" SET "ownerId" = "userId";
                                                                                                                                                                                                                                                                            UPDATE "Product" SET "ownerId" = "userId";
                                                                                                                                                                                                                                                                            UPDATE "ImportJob" SET "ownerId" = "userId";
                                                                                                                                                                                                                                                                            ```
